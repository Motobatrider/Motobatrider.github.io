Notes数据库的好坏

我从工作的第一家公司开始就用Notes数据库了，所以对它还蛮有感情的。Notes是典型的文档数据库，或者说文档数据库的概念就是Lotus提出的，不过总觉得那时候Lotus做这个是为了方便管理文档数据，所以在特定的场合用起来还是很方便的。

Lotus Notes是一个文档数据库管理系统：Notes数据库的最基本元素就是文档。这里的文档和关系数据库中的记录类似。Notes文档的结构是由表单（form）定义的，而表单由一组字段组成。

不过总觉得IBM对他们家这套系统也不是非常上心的样子，平心而论一整套Lotus的应用很方便，几个移动端做的也很有水准，不得不说看起来比微软的完成度更高，但是总有一种没认真推广让大家喜欢用的感觉。和我自己有时候做事的风格很像，所以还蛮亲切的。看网上总说MAC OS和WIndows的区别是设计师和程序员的区别，Lotus系统整个的程序员气息更为浓重，所以一开始用起来是会觉得非常不友好的。不过用熟了好处也出来了，就是我知道自己在干什么，1+1=2，有事情都在掌握之中的感觉。

修修补补的数据库

公司原来用的是一整套Lotus Notes解决方案，所以理所应当的我们的一整套出差请假报销系统都在Notes上面，现在赶时髦邮件和会议系统i上了O365,但是数据库什么的还在domino上，由于一些历史遗留问题，这些数据库多多少少都有些问题，有些是原始设定范围太广了，现在的用户范围可以精简了，有些当初就是应急开发的，很粗糙，还有些是开发的时候第三方公司有点捣浆糊，没怎么做好。另外有个问题是现在先不考虑中国的话，其他亚洲的国家几乎每个国家都有一套出差请假报销。我自己的Notes桌面上有将近20个各种行政系统的图标，非常的乱还难查数据。现在快8月了，到明年还有四个月，看看能不能改进一下。
本来这些都是找供应商做的，不过现在搞Notes开发的本来就不多，而且沟通起来也麻烦，有些公司还要先和客户经理说需求，然后再转告他们的工程师，一个圈子兜下来，基本上事情都做好了，反正现在手里可用的资源和模板都不少，索性自己来做做看。三四季度这个数据库改进的事和IT培训体系就是我工作的重点了。

这个任务的目标

  1. 改进现在亚太区其他国家(除中国外)的行政管理系统，简化操作，方便使用和管理。
  2. 争取统一亚太区的这些行政管理数据库并使各个数据库间资料的调用更简单
  3. 改进中国的整套数据库的方案，使访问和使用更简单
  4. 借这个机会和HR以及财务沟通好以后的流程，方便操作和管理

计划进度

  1. 整理手头资源，整理所有的亚太区的数据库的信息，看哪些可以合并
  2. 清理数据库模板，归纳做得比较好的组件
  3. 和各地人事和财务聊一下，整理需求
  4. 制定开发计划，改进数据库模板
  5. 开发新的数据库
  6. 测试
  7. 培训用户并上线
